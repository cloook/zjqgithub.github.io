redis的bitmap

2019-02-27

redis的bitmap使用场景

已经前提：

1. 数据在redis中都是二进制存储
2. setbit和getbit和bitcount是string数据类型的命令
3. 8bit表示一个ascll字符，因为是c写的redis
4. offset偏移量是从0开始

redis的bitcount计算给定key的字符串值中，被设置为1的比特位的数量

例如，c的二进制应该是01100011，所以bitcount计算出来结果是4

redis bitop命令

对一个或多个保存二进制位的字符串key进行位元操作，并将结果保存到destkey上

operation可以是 AND、OR、NOT、XOR这四种操作中的任意一种：

  1. BITOP AND destkey key [key …],对一个或多个key求逻辑并，并将结果保存到destkey。
  2. BITOP OR destkey key [key …],对一个或多个key求逻辑或，并将结果保存到destkey。
  3. BITOP XOR destkey key [key …]，对一个或多个key求逻辑异或，并将结果保存到destkey。
  4. BITOP NOT destkey key,对给定key求逻辑非，并将结果保存到destkey。

应用场景

1. 可以作为简单的布尔过滤器来判断用户是否执行过某些操作

2. 用户日活，月活，留存率的统计

案例：

  • 需求：每个用户登录/做任意操作，记为今天活跃，否则记为不活跃。计算出日活7日活月活等数据。
  • 具体措施：使用redis的bitmap
    • 设置一个key专门用来记录用户日活的，可以使用时间来翻滚比如1号的key为active01.
    • 使用每个用户的唯一标识映射一个偏移量，比如使用id，这里可以把id换算成一个数字或直接id的二进制值作为该用户在当天是否活跃偏移量
    • 用户登录则把该用户偏移量上的位置设置为1
    • 每天按日期生成一个位图(bitmap)
    • 计算日活则使用bitmap即可获得一个key的位值为1的量
    • 计算月活（一个月内登录的用户去重总数）即可把30天的所有bitmap做or计算，然后再计算bitcount
    • 计算留存率（次日留存=昨天今天连续登陆的人数/昨天登录的人数）即昨天的bitmap与今天的bitmap做and计算就是连续登陆的再做bitcount就得到连续登陆人数，再bitcount得到昨天登录人数，就可以通过公式计算出次日留存。

3. 实现用户上线次数统计

  • 假设现在我们希望记录自己网站上的用户的上线频率，比如说，计算用户A上线了多少天，用户B上线了多少天，诸如此类，以此作为数据，从而决定让哪些用户参加beta测试等活动 —- 这个模式可以使用setbit和bitcount来实现。
  • 比如说，每当用户在某一天上线的时候，我们就使用setbit，以用户名作为key，将那天所代表的网站的上线日作为offset参数，并将这个offset上的值设置为1
  • 举个例子，如果今天是网站上线的第100天，而用户peter在今天阅览过网站，那么执行命令setbit peter 100 1 ； 如果明天peter也继续阅览网站，那么执行命令setbit peter 101 1 ，以此类推
  • 当要计算peter总共以来的上线次数时，就使用bitcount命令：执行bitcount peter ，得出的结果就是peter上线的总次数

4. 用户在线状态及人数统计记录

同样是使用一个bitmap，以用户的id来映射偏移量，在线标识为1，下线标识为0.可以轻松实现用户上下线查询和总在线人数的统计

HashMap源码分析

2018-08-11

关于HashMap 自己的回顾

数据结构原型：

数组+链表

先根据hash算法，得出hash值，放在 Node[] 中，当 hash值相同时，往下依次排列，当单链表长度达到一定值时，单链表转成红黑树（jdk 1.8）。

计算hash值：

Object.hashCode() 对 Node[] 的size 取模
源码里采用位运算，效率更高
hashMap 默认长度是 1<<4 = 2^4 = 16
所以取模运算等同于 hash & (16-1)
可以得出 0-15的索引值，即Node[]的索引位置

& 运算

111010101001111
&
01111


结果一定是在 00000~01111之间，即0-15

考虑到哈希碰撞结果的不稳定性，hash算法进行优化

Object.hashCode()^Object.hashCode()>>>16

hashCode的高16位和低16位进行与或运算，是一个合适的散列分布
为什么是16呢，因为hashCode是int
拿这个结果再 & (Node[].size()-1) 得出的索引值是比较均匀合理的

Node[] 扩容

每次计算完哈希值之后，去Node[] 排坑，如果坑位还没有被占用，就填进去，顺便 ++占位值；
当占位置 > Node[].size() * 默认科学计量值(0.75)的时候，就该扩容了

扩容每次扩充两倍，map的size有上限，最大值是 2^32
为什么是2倍呢？
这还是回到哈希算法中，我们用到了 & 运算来计算最后的hash值，也就是Node[]的size一定要是2的n次幂才是合适的
为什么是2的n次幂呢？
因为任何2的n次幂一定是 1000… 这种结构的然后 Node[].size() -1 就是 0111… 这种结构的
0111…这种结构才能够保证Node[]数组的每一个索引值都有可能被用到，不会造成空闲位置的资源浪费

Node[] 扩容之后

Node[] 扩容之后，要重新计算hash值，对已存在的Node进行迁移
Node[]上任意节点的迁移一定是原地不动或者 index+扩充长度
为什么这样呢，
因为我们扩充2倍之后，等于加了一位，比如原来是16 = 10000，现在变成了 32 = 100000 在重新计算hash值的时候，& 运算由01111 变成了 011111
  01111
011111
可以看出 后面四个1保证了后四位&运算结果不会变化，然后再往左一位结果可能是0也可能是1，如果是0那就是原地不动，如果是1，那就是index+扩展长度

命令集

centos 查看所有运行中的服务的详细信息命令：

netstat -tunpl

停止服务：

pkill -9 (服务名称，如 nginx,docker…)

服务器文件复制:

scp -r dir service:dir

mysql> set password for 用户名@localhost = password(‘新密码’);

nohup java -jar helloworld.jar &

nohup的意思不挂服务，常驻的意思，除非云服务器重启，那就没法了；最后一个&表示执行命令后要生成日志文件nohup.out

centOS实践

2018-03-06

安装java

用yum安装JDK

1.查看yum库中都有哪些jdk版本(暂时只发现了openjdk)
yum search java|grep jdk

2.选择版本,进行安装
yum install java-1.7.0-openjdk
安装完之后，默认的安装目录是在: /usr/lib/jvm/java-1.7.0-openjdk-1.7.0.75.x86_64

3.设置环境变量
vi /etc/profile
在profile文件中添加如下内容

source /etc/profile

4.验证JDK有效性
java -version

安装mysql

1.下载mysql的repo源
wget http://repo.mysql.com/mysql-community-release-el7-5.noarch.rpm

2.安装mysql-community-release-el7-5.noarch.rpm包
sudo rpm -ivh mysql-community-release-el7-5.noarch.rpm
安装这个包后，会获得两个mysql的yum repo源：/etc/yum.repos.d/mysql-community.repo，/etc/yum.repos.d/mysql-community-source.repo

3.安装mysql
sudo yum install mysql-server

4.重置mysql密码
mysql -u root
登录时有可能报这样的错：ERROR 2002 (HY000): Can‘t connect to local MySQL server through socket ‘/var/lib/mysql/mysql.sock‘ (2)，原因是/var/lib/mysql的访问权限问题。下面的命令把/var/lib/mysql的拥有者改为当前用户：
sudo chown -R root:root /var/lib/mysql
重启mysql服务
service mysqld restart
接下来登录重置密码

安装nginx

直接使用 yum 来安装 Nginx
yum install nginx -y
使用 nginx 命令启动 Nginx
nginx
此时，访问 http://<您的域名> 可以看到 Nginx 的测试页面
